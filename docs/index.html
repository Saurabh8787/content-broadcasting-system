Content Broadcasting System

Backend API for educational content broadcasting — teachers upload, principals approve, students view live.

https://content-broadcasting-system-1-xdm1.onrender.com

Stack

Node.js · Express · PostgreSQL · JWT · bcrypt · Supabase Storage

Auth

JWT Bearer token. Include Authorization: Bearer <token> on all protected routes.

Roles

principal — approves/rejects content
teacher — uploads content

File Upload

Accepted: JPG PNG GIF
Max size: 10MB

Authentication
How to authenticate:
1. Call POST /auth/login with your credentials
2. Copy the token from the response
3. Add header Authorization: Bearer <your_token> to all protected requests

POST /auth/register Register a new account Public

Create a new teacher or principal account.

Request Body (JSON)
FieldTypeRequiredDescription
namestring✅Full name
emailstring✅Email address
passwordstring✅Min 4 characters
rolestring✅teacher or principal

Content — Teacher
POST /content/upload Upload new content 🔒 Teacher

Upload an image file with scheduling details. Content starts as pending and must be approved by the principal before going live.

Note: Content without start_time and end_time will never appear in the live feed even after approval.
Request Body (multipart/form-data)
FieldTypeRequiredDescription
titlestring✅Content title
filefile✅JPG, PNG or GIF — max 10MB
subjectstring✅e.g. maths, science
descriptionstring❌Optional description
start_timestring❌ISO 8601 e.g. 2026-04-27T00:00:00Z
end_timestring❌ISO 8601 e.g. 2026-04-28T23:59:00Z
rotation_durationinteger❌Minutes per rotation slot (default 5)
Example Response
{
  "success": true,
  "message": "Content uploaded successfully. Pending approval.",
  "data": {
    "id": 3,
    "title": "Algebra Chapter 1",
    "subject": "maths",
    "status": "pending",
    "file_url": "https://xxx.supabase.co/storage/v1/object/public/content-uploads/content-123.png",
    "start_time": "2026-04-27T00:00:00.000Z",
    "end_time": "2026-04-28T23:59:00.000Z",
    "rotation_duration": 5,
    "created_at": "2026-04-27T09:00:00.000Z"
  }
}
Responses
201Content uploaded — pending approval
400Invalid file type, file too large, or missing required fields
401Unauthorized
403Teachers only

Broadcasting — Public
GET /content/live/:teacherId 🔴 Get live content Public

Public student-facing endpoint. Returns the currently active content for a teacher based on the rotation schedule. No authentication required.

Scheduling Algorithm:
position = (NOW - slot_epoch) % total_cycle_seconds
Each teacher's subjects rotate independently in a continuous loop.
Path Parameters
ParamTypeDescription
teacherIdintegerTeacher's user ID
Query Parameters
ParamTypeDescription
subjectstringFilter by subject (optional)
Example — Content Available
{
  "success": true,
  "data": {
    "available": true,
    "teacher": "Teacher One",
    "items": [
      {
        "subject": "maths",
        "id": 3,
        "title": "Algebra Chapter 1",
        "description": "Introduction to algebra",
        "file_url": "https://xxx.supabase.co/storage/v1/object/public/content-uploads/content-123.png",
        "file_type": "image/png",
        "start_time": "2026-04-27T00:00:00.000Z",
        "end_time": "2026-04-28T23:59:00.000Z",
        "rotation_duration_minutes": 5
      }
    ],
    "retrieved_at": "2026-04-27T10:00:00.000Z"
  }
}
Example — No Content
{
  "success": true,
  "data": {
    "available": false,
    "message": "No content available.",
    "items": []
  }
}
Responses
200Live content returned (or available=false if none)
